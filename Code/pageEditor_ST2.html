Home


Technical Description


For the Home page, I implemented a modern, responsive design using several key HTML and CSS techniques:


HTML


  • Hero Section: I Created with a full-width video background using the <video> tag with autoplay, muted, and loop attributes. Text content is overlaid using absolute positioning with h2 and h3 heading.

  • Content Organization: I Used distinct <section> elements with unique IDs (#targets, #news, #programs, #events) to create clearly defined content areas and enable smooth navigation.

  • Responsive Layouts: Implemented CSS Grid and Flexbox through class-based styling (.target-grid, .news-grid, .program-content, .events-grid) to create adaptable content blocks that respond to different screen sizes. for that part I used @media tags.

  • Structured Content: The programs section features nested unordered lists for organized content presentation.

  • Card-Based Design: Used consistent styling through reusable classes (.target-card, .news-card, .event-card) to maintain visual represntation while effectively separating different types of information.

CSS


  • Hero Section: Built a full-width hero area with video background using position: absolute and transform: translate(-50%, -50%) to center the video perfectly behind the text content.

  • Card Layouts: Used CSS grid with display: grid and gap: 30px to create flexible card arrangements that adapt to different screen sizes through the minmax() function.

  • Visual Effects: Added subtle interactive elements like button hover states with transform: scale(1.03) and color transitions to improve user engagement and highlight clickable items.

  • Section Design: Applied consistent padding, text alignment, and spacing to create clear visual sections that guide users through content.

  • Background Styling: Used background images with overlay effects in the news section, along with backdrop-filter: blur(10px) to create depth and visual interest.

  • Flexible Content: Created a program section with display: flex to position text and images side-by-side on larger screens while stacking them on mobile views.

  • Mobile Optimization: Implemented three breakpoints (1024px, 720px, 480px) to adjust font sizes, spacing, and layouts for optimal viewing across devices.

  • Image Control: Used object-fit: cover to ensure images maintain proper proportions while filling their containers completely.

Accessibility


  • Semantic HTML Structure: Used appropriate heading hierarchy (h1, h2, h3) throughout the page to create a logical document outline that screen readers can navigate effectively.

  • Responsive Design: Created flexible layouts that adapt to different screen sizes.

  • ARIA Landmarks: Implemented ARIA landmarks (main, section, nav) to help screen reader users navigate between different content sections more efficiently.

Table


Technical Description


For the Table page, I implemented a structured and informative display of conservation data using essential HTML and CSS techniques:


HTML


  • Table Structure:I Used the <table> element with table tags (<tr>, <th>, <td>) to organize data in a clear, accessible format.

  • Advanced Table Features: Implemented colspan and rowspan attributes to create a hierarchical header structure and merge cells for related conservation efforts, improving data presentation and readability.

  • Content Organization: I Created a container <div> with class "table_wrapper" to ensure proper table display across different devices while maintaining structural unity.

  • Visual Elements: Incorporated <img> tags with appropriate alt attributes for each animal species, enhancing the visual appeal while maintaining accessibility.

  • Information Hierarchy: Used proper heading (<h1>) for the table title "Global Forest Cover and Conservation Efforts" to establish context and improve page organization.


CSS


  • Container Structure: Created a .container class with 90% width and auto margins to center content, add a padding and background color for better readability.

  • Table Responsiveness: Implemented a .table-wrapper with overflow-x: auto to ensure tables remain accessible on smaller screens while maintaining structural integrity.

  • Visual Enhancement: Applied border-radius, box-shadow, and backdrop-filter properties to the table element, creating a modern appearance depth.

  • Image Integration: Used CSS to control table images with object-fit: cover and customized dimensions, ensuring consistent image display while adding interactive hover effects with transform: scale(1.1).

  • Color Scheme: Established a color palette using forest-inspired tones (#2c362c, #d0d0d0) for table elements, while maintaining readability.

  • Interactive Elements: Added hover states to table cells using td:hover with background color transitions and font-weight changes to improve user interaction and content emphasis.

  • Media Queries: added responsive breakpoints at 768px to adjust heading sizes, container padding, and cell dimensions for optimal display across devices.

  • Typography Hierarchy: Used varying font sizes, letter-spacing, and color contrast between headers and data cells to establish clear information hierarchy and improve scannability.

Accessibility


  • Table Structure: Used proper table elements (th, td) with appropriate colspan and rowspan attributes

  • Image Alternatives: Provided detailed alt text for animal images that included both species name and conservation status information.

  • Visual Design:I used adequate color contrast in some areas (dark text on light backgrounds) and hover effects provide visual feedback when interacting with table cells

Content Page


Technical Description


HTML


  • Content Structure: Used <div> elements with meaningful class names (top, why, how, join) and corresponding IDs to organize content logically and enable smooth internal navigation.

  • Internal Navigation: Implemented an internal_link section with anchor links using <a href="#section-id"> to create a user-friendly way to jump between major content sections.

  • Interactive Elements: Added a "go up" button with integrated SVG icon and an "Apply" button in the join section to enhance user engagement and provide clear calls to action.

  • Visual Hierarchy:I Used proper heading elements (<h1>, <h2>) with appropriate sizing to improve content readability.

  • External Resource Management: Linked content-specific and global CSS files to maintain styling consistency while properly separating design concerns.

CSS


  • Base Layout: Set up a full-height body with dark background color (#444544) and consistent font family to create a solid foundation for the conservation theme.

  • Hero Section: Built a striking top section with a background nature image, large text (6em for h1), and semi-transparent content area using backdrop-filter:blur(50px) for better text readability.

  • Color Scheme: Used green accents (#81cd34, #529F25) for headings and interactive elements to reinforce the environmental theme while maintaining good contrast with light text.

  • Navigation Links: Created a flexible internal link menu with display:flex and justify-content:space-evenly that wraps content on smaller screens while adding subtle hover effects.

  • Section Spacing: Applied consistent margins (50px) between content sections to create visual breathing room and improve content separation.

  • Interactive Elements: Added hover effects to buttons and links using color changes and transform: scale() properties to provide visual feedback to users.

  • Helper Features: Included a fixed-position "go up" button with position: fixed in the bottom-right corner to help users navigate long content easily.

  • Mobile Design: Created two breakpoints (800px and 480px) that adjust text sizes, padding, and layout for smaller screens, including a switch from flex to block display for internal links on very small devices.

Accessibility


  • Content Structure: Clear hierarchical structure with headings (h1, h2) and I also used inernal page navigation links.

  • Visual Design: added hover effects for better viusal represntation. also used margin to create space between content sections.

Challenges and Lessons Learned


When I was making the Life on Land website, I ran into couple of problems. First time reading CW specification it was hard to keep track on every details. So I took notes about key points. Acording to the topic I knew theme should be somewhat green, but was struggling find the right color palette. After discussing it with team memmbers we came identified a color palette which added better readability.
what looked good on my laptop was huge on phones! I had to create different sizes for different screen sizes. The "go up" button kept causing weird issues on mobile browsers until I messed around with the positioning values.

Through all this, I learned that I should've tested on different screen sizes way earlier, and that using em units for text sizes helps everything scale better. Also, sticking to the same green colors throughout really helped make the whole site feel connected to the conservation theme.

Compliance


The Life on Land website was built following modern web standards and accessibility best practices. W3C Validator was used to ensure clean, error-free HTML and CSS. To meet WCAG guidelines, we used semantic HTML, proper form labeling, alt text for images, and sufficient color contrast. Responsive design was optimized for different screen sizes, with em units for better scalability. Navigation elements, like the "go up" button, were adjusted for smooth functionality across devices. By maintaining a consistent color scheme and optimizing performance, the site ensures accessibility, usability, and a seamless user experience.
