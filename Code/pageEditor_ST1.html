Splash Page

Technical Description

  • 1. splash.html (Splash Screen Page)

    • <!DOCTYPE html>: Declares the document type as HTML5.
    • <html lang="en">: Specifies English as the document language, improving accessibility.
    • <head> Section:
      • <meta charset="UTF-8">: Defines character encoding to UTF-8 for proper text rendering.
      • <meta name="viewport" content="width=device-width, initial-scale=1.0">: Ensures responsiveness by setting the viewport width to the device width.
      • <meta http-equiv="refresh" content="4;url=Volunteer.html">: Redirects users to Volunteer.html after 4 seconds.
      • <link rel="stylesheet" href="assets/css/splash.css">: Connects the page to an external CSS file for styling.
      • <title>TerraBloom</title>: Sets the title of the webpage.
    • Body Content
      • Background Video (<video>): The <video> element creates an immersive background.
        • autoplay: Starts playing automatically.
        • muted: Ensures no sound.
        • loop: Repeats indefinitely.
        • <source>: Provides the video file (splash.mp4) in MP4 format.
        • Fallback text ensures accessibility for browsers that do not support <video>.
      • Content Container (<div class="splash-container">): Contains both the video and the overlay panel.
      • Overlay Panel (<div class="panal">): Displays branding elements on top of the video.
        • Includes a logo (<img>), the title (<h1>), a mission statement (<h3>), and team members (<p>).
      • Loading Animation (.loader): Uses five <div class="dot"> elements.
        • Each dot has an animation delay for a smooth transition effect.

  • 2. splash.css (Splash Page Styling)

    • Global Reset and Structure
      • * { margin: 0; padding: 0; box-sizing: border-box; }: Removes default margins/padding for consistency. Uses box-sizing: border-box; to ensure width/height includes padding and borders.
      • body, html: width: 100%; height: 100%; overflow: hidden; → Prevents scrolling. display: flex; justify-content: center; align-items: center; → Centers content. font-family: Arial, sans-serif; → Ensures a clean, readable font.
    • Background Video Styling (video)
      • position: absolute; top: 0; left: 0; width: 100%; height: 100%; object-fit: cover; → Covers the entire viewport while maintaining aspect ratio.
    • Overlay Panel (.panal)
      • position: absolute; top: 0; left: 0; width: 100%; height: 100%; background: rgba(0, 0, 0, 0.5); → Semi-transparent dark background overlay improves text readability.
      • display: flex; flex-direction: column; justify-content: center; align-items: center; → Uses flexbox for a centered layout.
      • color: white; text-align: center; → Ensures text contrasts well against the dark overlay.
    • Typography Styling
      • .webTitle { font-size: 4em; } → Large, prominent site title.
      • .mission { width: 50%; } → Limits mission text width for readability.
      • .team { font-size: 2em; font-weight: bold; margin-top: 20px; } → Highlights team names.
    • Loading Animation (.loader and .dot)
      • .dot: width: 12px; height: 12px; background: white; border-radius: 50%; → Circular dots.
      • animation: dot-wave 1.5s infinite ease-in-out; → Continuous bouncing animation.
      • Keyframe Animation (@keyframes dot-wave): Animates each dot to move up and down (translateY). Opacity changes enhance visibility effects. Different animation-delay values stagger movements for a wave-like effect.

  • 3. mainStyle.css (Global Website Styling)

    • Navigation (nav)
      • font-weight: 500; background-color: #66A82C; box-shadow: 3px 3px 3px rgba(0,0,0,0.1); → Ensures a green navigation bar with a subtle shadow.
      • Navigation Links (nav a): text-decoration: none; color: #fff; → Removes underlines and ensures white text. display: flex; align-items: center; → Centers text vertically.
      • Hover Effect (nav a:hover): Changes color for better interactivity.
    • Sidebar Menu (.sidebar)
      • display: none; position: fixed; width: 250px; height: 100vh; background-color: rgba(102, 168, 44, 0.27); backdrop-filter: blur(20px); → A hidden sidebar that appears on mobile.
      • Media Queries for Responsiveness (@media)
        • @media (max-width: 700px) { .hideOnMobile { display: none; } } → Hides navigation links on smaller screens.

  • 4. Footer Styling (.footer-main and .footer-bottom)

    • Grid Layout (.footer-main): display: grid; grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)); gap: 3.5rem; → Creates a flexible footer layout that adapts to screen sizes.
    • Footer Text (.footerContent): color: #fff; font-size: 1em; → Ensures readable contrast.
    • Footer Links (.footerContent li a): transition: all 0.4s ease; → Smooth scaling effect on hover.
    • Copyright Bar (.footer-bottom): height: 40px; background-color: #66A82C; color: #fff; → Maintains consistent branding.

Accessibility

  • Color and Contrast
    • Text is white on a semi-transparent black background, ensuring sufficient contrast.
    • Loader dots have good contrast with the background.
  • Responsive Design
    • Viewport meta tag ensures proper scaling on different devices.
    • Flexbox layout centers content responsively.
  • Animation and Motion
    • Loader animation is subtle and not overly distracting.
  • Text Resizing
    • Font sizes use relative units (em), allowing users to resize text without breaking the layout.
  • Video Accessibility
    • Video is set to autoplay, muted, and loop to reduce disruption for users with auditory sensitivities.
    • Improvement suggestion: Add captions if the video contains important information.

Accessibility

  • 1. Semantic HTML

    This topic covers using proper HTML tags like <header>, <nav>, <section>, <article>, and <footer> to create a well-structured page. Using correct headings (<h1>, <h2>, etc.) improves accessibility, especially for screen readers. Semantic HTML helps assistive technologies understand the layout and purpose of the content.

  • 3. Color Contrast and Colorblind Accessibility

    Ensuring that your website has a high contrast between text and background (e.g., dark text on light backgrounds) is critical for users with low vision or color blindness. It’s important to avoid relying on color alone to convey information. This can be achieved by combining color with text labels, icons, or patterns.

  • 5. Alt Text for Images

    Alt text should be used for all images to provide a text alternative for screen readers. This is essential for users who cannot see the images. Descriptive alt text ensures that users understand the content and context of images, including icons, banners, or any visual elements.

  • 6. Responsive Design

    A responsive design ensures that your website looks good and functions well on devices of all sizes. This includes mobile, tablet, and desktop. It involves using flexible grid layouts, resizing images, and making sure elements adapt to various screen sizes. It also ensures that the site is accessible for people who may use screen magnifiers or other assistive technologies.

Compliance

  • WCAG (Web Content Accessibility Guidelines) Compliance
    • Text Contrast & Color Accessibility:
      • Ensure the text color contrasts well with background colors. Use tools like the WebAIM Contrast Checker.
      • Example: Heading text in .section-heading1 has good contrast with the dark green (#2c362c) against a white background.
  • Mobile Responsiveness Compliance
    • Responsive Design:
      • Ensure the website adapts to different screen sizes using fluid layouts (e.g., CSS Grid, Flexbox).
      • Media queries should be used to adjust the layout on smaller devices (mobile, tablet, etc.).
    • Mobile-Friendly Navigation:
      • Ensure that the sidebar and other navigation elements are mobile-friendly, such as using a hamburger menu for smaller screens.
  • HTML Semantics Compliance
    • Semantic HTML:
      • Use semantic HTML tags like <header>, <footer>, <article>, and <section> for clear document structure.
      • Proper use of heading tags (<h1>, <h2>, etc.) for proper document hierarchy.
  • Browser Compatibility
    • Cross-Browser Testing:
      • Ensure that your website functions well across all modern browsers, including Chrome, Firefox, Safari, and Edge.
      • Test for mobile and tablet browsers to ensure good usability.
  • Performance Compliance
    • Speed Optimization:
      • Compress images and videos to improve page load time.
  • Data Privacy Compliance
    • GDPR Compliance:
      • If collecting personal data (e.g., emails), ensure that your site complies with GDPR, including a privacy policy and consent options.
