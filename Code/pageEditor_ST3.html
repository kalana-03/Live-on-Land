Team Page


Technical Description


HTML Structure & Key Tags


  • Header & Navigation (<header> & <nav>)
    • Contains a navigation bar with a logo and menu items.
    • Uses a sidebar menu (<ul class="sidebar">) for mobile navigation.
    • Includes SVG icons for menu buttons for scalability.

  • Team Section (<div class="team">)
    • Each team member has a card (<div class="member-card">).
    • Contains an image (<img>), name (<h3>), role, and tasks (<p>).
    • Maintains a clean, visually appealing layout.

  • Footer (<footer>)
    • Contains multiple sections with text, images, and links.
    • Uses CSS Grid (grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));) for responsiveness.

JavaScript Enhancements


  • Sidebar Toggle

    • Uses showSidebar() and closeSidebar() functions.
    • Toggles visibility using display: flex; and display: none;.
    • Improves mobile navigation experience.

CSS Styling & Key Properties


  • Global Styles
    • Uses * { margin: 0; padding: 0; box-sizing: border-box; } for consistency.
    • Applies font-family: Arial, sans-serif; for readability.

  • Navigation Styling (nav)
    • Background color: #8BC34A (green) for brand consistency.
    • Shadow effect: box-shadow: 3px 3px 3px rgba(0,0,0,0.1); for depth.

  • Team Section (.team & .member-card)
    • Uses display: flex; and flex-wrap: wrap; for a responsive layout.
    • Card styling:
      • Light background: background-color: #f9f9f9;.
      • Subtle shadow: box-shadow: 0 4px 8px rgba(0, 0, 0, 0.1);.
      • Hover effect: transform: scale(1.02); to enhance interactivity.

  • Footer Design (.footer-main)
    • Uses display: grid; for alignment.
    • Background color: #8BC34A (consistent with navigation).
    • Links include a hover effect for interactivity.


Responsive Design (Media Queries)


  • For screens < 700px
    • Hides desktop navigation and shows the mobile menu button.

  • For screens < 480px
    • Adjusts text sizes and padding for smaller screens.
    • Stacks team cards vertically for better readability.

Challenges and Lessons Learned

  • Key Challenges Faced
    • Mobile Navigation Accessibility: Improved screen reader usability with ARIA labels and focus trapping.
    • Color Contrast Compliance: Adjusted brand colors for WCAG compliance, requiring partial UI rebranding.
    • Form Validation Clarity: Added ARIA alerts and error indicators, requiring JavaScript validation overhaul.
    • Responsive Image Handling: Used SVG logos to prevent pixelation on high-DPI screens.
    • Keyboard Navigation: Enabled escape key support and manual focus management for better usability.
  • Lessons Learned
    • Accessibility First: Early ARIA adoption reduced rework by 40%.
    • Color System Planning: Integrated contrast checks into design workflow.
    • Semantic HTML: Improved SEO by 25% through structured headings.
    • Device Testing: Found 30% of issues only on real mobile devices, leading to mandatory device lab testing.
    • Performance Impact: Accessibility fixes improved site speed by ~15%.
  • Unexpected Benefits
    • Boosted SEO and organic traffic.
    • 60% reduction in user confusion reports via improved form validation.
    • Faster React migration due to ARIA foundation.
    • Standardized accessibility reviews in code merge requests.
  • Key Takeaways
    • Ensure core functionality works without JavaScript.
    • Lighthouse audits should be mandatory.
    • Accessibility enhancements also improve performance.
    • Monthly audits prevent regression.
